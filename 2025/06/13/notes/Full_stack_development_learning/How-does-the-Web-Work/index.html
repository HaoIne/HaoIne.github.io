How does the internet work?

Here is some notes for a Mozilla article on “How does the Internet work?”

Now I know how the information goes from my laptop to the web server and then come back.

  1. Information comes from my laptop to the switch (交换机) in my home, then switch forwards it to the router (路由)

  2. Router sends the information to modem (调制解调器), which could turn the information from electrical signal to optical signal, then the information could travel through telephone infrastructure, arrive to the ISP (Internet Service Provider).

  3. ISPs manages some special routers, which are connected to each other, and connected to the global Internet. Upon catching the information your laptop sent, the IPS use DNS (Domain Name System) to search for the IP address of the website you are request.

  4. ISP then send your information to that IP address through routers.

The difference between web page, website, web server, and search engine

See this

Web Page

Web Page is a simple file, often a HTML file, which could be displayed by a browser. A web page could embed a variety of different types of resources like:

  • Style information - controlling a page’s look-and-feel
  • Scripts - which add interactivity to a web page
  • Media - images, sounds, and videos

Note: Browsers can also display other documents such as PDF files and other resources such as images or videos, but the term web page specifically refers to HTML documents.

Website

In a word, Website is a sort of Web Page, which are linked with each other, and share a unique domain name.
In a Website, you could jump from a web page to another from the explicit link in the web page.

Note: It is also possible to have a single-page app: a website that consists of a single web page that is dynamically updated with new content when needed.

Web Server

A web server is a computer hosting one or more websites. “Hosting” means that all the web pages and their associated files are available on that computer. The web server will send web page files it is hosting to a user’s browser when they attempt to load it.

Note: A web server could host one or more websites, each has it’s own domain name.

Search Engine

Actually, search engine is also a web service, a program to search for websites with domain names or IP addresses.

How does the web works?

Watch this article

In order for data to go back and forth between your computer and the web server, we need the following thins:

  • Your Internet Connection: Internet is like the road, which allow data to go through.

  • TCP/IP: Transmission Control Protocal and Internet Protocal are communication protocols that define how data should travel across the internet. This is like the transport mechanisms.

  • DNS: The Domain Name System is like an address book for websites. Browsers could know an IP address of website through it’s domain and DNS.
    A video about DNS

  • HTTP: Hypertext Transfer Protocal is an application protocal that defines a language for clients and servers to speak to each other.

  • Files: A website is composed of many different files, These files come in two main types:

    • Code: Websites are built primarily from HTML, CSS, and JavaScript codes, which the browser interprets and assembles into a web page to display to a user.
    • Assets: This is a collective term for all the other items that appear on a website - such as images, music, video, Word, and PDFs etc, everything except codes.

What exactly happens?

  1. When you type a domain name in the address bar of your browser, browser goes to the DNS server and finds the real address of the server tha the website lives on.

  2. The browser send an HTTP request message to the server, across your internet connection using TCP/IP.

  3. The server sends the client a HTTP response, which contains the home page file of the website, as a series of small chunks called data packets.

  4. The browser assembles the small chunks into a complete web page and displays it to you, then you see an entire web page on your browser.

Components of a URL

URL is Uniform Resource Locator, defining the location of a unique resource on the internet.

like this one

The main parts of the URL are:

  • https
    The protocol being used to send the request. In this case, we are using HTTPS, which is a secure version of HTTP.

  • developer.mozilla.org
    The domain name of the URL, which represents the top-level location of the server you are connecting to.

  • /en-US/
    The path to the resource on the server that you are accessing. MDN keeps all its US English content in a folder called en-US, which is what this URL is pointing to.
